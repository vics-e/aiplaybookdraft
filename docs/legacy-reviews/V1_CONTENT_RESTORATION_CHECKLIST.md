# V1 Content Restoration Checklist

## Short Summary

The original source playbook in `src/imports/ai-playbook-accountants.txt` has 66 pages.

The current live website content in `src/app/data/playbookData.ts` has 32 entries, including the cover, contents page, playbook pages, final page, and activity summary.

Some extra playbook-style content exists in unused component files such as `Section1.tsx` through `Section7.tsx`, `Hero.tsx`, and `TableOfContents.tsx`. That content is not part of the live routed playbook unless it is migrated into `playbookData.ts`, rendered by `PageContent.tsx`, or otherwise connected to `SageAIPlaybook.tsx`.

The V1 content task is not to recreate 66 separate website pages. Merging source pages into fewer website pages is acceptable. However, every source page, activity, template, example, and important instruction must be represented somewhere in the live website flow.

## Content Restoration Strategy

Merging pages is acceptable for V1. The website does not need to have 66 separate screens, and a shorter live route can still be faithful if every source page is represented clearly.

Every source page's content must be represented in the live website. If a page is merged, the merged destination should still include the source page's key explanation, examples, instructions, and activity. A source page should not disappear just because its topic is mentioned briefly elsewhere.

Content that exists only in unused `Section1.tsx` through `Section7.tsx` files should not be treated as complete. It should be marked as "Exists in repo but not live" until it is connected to the live app or migrated into `playbookData.ts` and rendered by `PageContent.tsx`.

Activities should be preserved unless there is a deliberate decision to rewrite them. If an activity is rewritten, the replacement should capture the same user task, not just a simpler reflection.

Templates can be added as pages, expandable sections, or resource blocks. For V1, the safest approach is to add them into the live `playbookData.ts` flow as Section 7 resource entries, then make sure `ActivitySummary.tsx` can still show any user answers that should appear in the final summary/download/print view.

Recommended implementation sequence:

1. Restore missing Section 2 guardrails first, because they are professional risk controls.
2. Restore missing templates/resources in Section 7, because they support multiple earlier sections.
3. Restore missing pricing and 90-day plan pages, because those are large content gaps.
4. Expand shortened live pages using the source text and, where useful, unused component content.
5. Review every activity in `playbookData.ts` against the source playbook and add missing answer fields.
6. Check `ActivitySummary.tsx` after activity restoration so the final summary captures the restored activities.
7. Only then review page order/navigation in `SageAIPlaybook.tsx` and live rendering in `PageContent.tsx`.
