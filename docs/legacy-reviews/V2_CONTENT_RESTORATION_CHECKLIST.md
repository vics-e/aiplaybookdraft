## Content Restoration Strategy

Merging pages is acceptable. The website does not need 66 separate pages, but every source page's content must still be represented in the live website.

Content that exists only in unused `Section1.tsx` to `Section7.tsx` files should not be treated as complete until it is connected to the live app or migrated into `playbookData.ts` and rendered by `PageContent.tsx`.

Activities should be preserved unless deliberately rewritten. If an activity is rewritten, the replacement must capture the same user task and answer detail. For example, a source open reflection should not be reduced to a yes/no radio control unless another field captures the reasoning.

Templates can be added as pages, expandable sections, or resource blocks. For V1, the safest approach is to add them as Section 7 entries in `playbookData.ts`, then ensure `ActivitySummary.tsx` can display any new answer formats.

Recommended restoration order:

1. Restore missing professional guardrails in Section 2.
2. Restore missing Section 7 templates/resources, because they support earlier sections.
3. Restore changed activities on already-live pages, especially task counts, question wording, and template fields.
4. Restore missing pricing and 90-day plan pages.
5. Expand shortened pages using source text first, then reuse unused component content only after source verification.
6. Review all page titles against the source titles.
7. Run the app and check that navigation, activities, summary, print/download, and responsive layout still work after new entries are added.
